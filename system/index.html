Systemprinsipp

NORMIA etablerer entydig sammenheng mellom krav, utførelse og dokumentasjon. Ingenting anses som levert før det er definert, utført, kontrollert, dokumentert og arkivert.

Objekt- og ID-logikk

Alle elementer behandles som objekter med entydig identitet, ansvar og status. ID-en er bærer av sporbarhet og binder sammen prosjektering, utførelse, kontroll og FDV.

Eksempel: NORMIA-EKS-0001-O-04 (objekt) → …-KON-004 (kontrollpunkt) → …-DOK-PR-001 (dokument).

Dokumenthierarki

Dokumentasjon er strukturert og knyttet til objekter og kontrollpunkter. Dokumenter uten definert tilknytning anses ikke som gyldig dokumentasjon.

  • Prosjektering: kravgrunnlag, valg, forutsetninger
  • Utførelse: arbeidsrekkefølge, kritiske punkter, egenkontroll
  • Kontroll: verifikasjon mot kriterier, avvik og lukking
  • FDV: overlevering, drift og revisjonssikker historikk

Revisjon og endringskontroll

Endringer håndteres gjennom formell revisjon. Tidligere revisjoner slettes ikke, men arkiveres som historikk. Gjeldende versjon skal alltid være entydig.

Se Revisjoner for endringshistorikk.

Status og sporbarhet

Status spores til tidspunkt, ansvarlig rolle og dokumentasjon. Systemet muliggjør presise svar på: hva, når, av hvem og på hvilket grunnlag.

Systemlogikk: Objekt → Dokument → Kontrollpunkt → Status → Arkiv
