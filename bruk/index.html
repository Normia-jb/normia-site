Bruk

Denne siden viser hvordan NORMIA brukes i praksis – fra oppstart til endring. Følg rekkefølgen. Ikke hopp over steg.

1

Start et nytt prosjekt

Start alltid med å avklare hvilke deler av NORMIA som gjelder for prosjektet:

  • Hvilke metodikker brukes (PRO / UF / KON / FDV)?
  • Hvilke standarder er styrende?
  • Hva er prosjektets avgrensning?

Prosjektspesifikke forhold dokumenteres – metodikken endres ikke.

2

Bruk metodikken som fasit

Metodikk-sidene er styrende dokumenter. Utførelse, kontroll og FDV skal alltid kunne spores tilbake til:

  • Dokument-ID
  • Versjon
  • Gjeldende status

Hvis noe ikke er beskrevet i metodikken, er det ikke definert.

3

Dokumenter prosjektets valg

Alle prosjektspesifikke beslutninger dokumenteres som:

  • Prosjekteringsavklaringer
  • Avvik
  • Tillegg eller presiseringer

Ikke endre metodikken for å “tilpasse prosjektet”.

4

Utfør og kontroller

Utførelse og kontroll følger metodikken punkt for punkt:

  • Egenkontroll før lukking
  • Kontroll mot definerte kriterier
  • Sporbar dokumentasjon per objekt/sone

Det som ikke kan dokumenteres, anses ikke som utført eller kontrollert.

5

Revider ved endring

Når noe endres:

  • Vurder om endringen påvirker praksis
  • Opprett ny versjon hvis nødvendig
  • Loggfør hva som er endret og hvorfor

Endringer uten revisjon regnes som avvik.
